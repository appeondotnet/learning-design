# 模板方法模式
在一个方法中定义一个算法的骨架，而将一些步骤延迟到子类中。模板方法使得子类可以在不改变算法结构的情况下，重新定义算法中的某些步骤。

## 思考方式
泛化，定义原语操作

## 优点
* 超类主导了算法，而且保护了算法
* 算法只存在与一个地方，容易修改
* 提供了一个框架，可以让新的实现快速接入进来，只需要实现自己的方法即可
* 超类提供算法本身，子类复杂提供完整实现

## 钩子
钩子是一种被声明在抽象类中的方法，但只有空的或者默认的实现，可以让子类有能力对算法的不同点进行挂钩，要不要挂钩，由子类决定。

## 何时使用钩子
当子类必须提供算法中某个方法或步骤实现时，使用抽象方法。  
如果算法的这部分时可选的，就用钩子，以为子类可以选择实现这个钩子，但并不强制这么做。

## 与策略模式
* 模板方法定义了一个算法大纲，子类只定义其中的个别步骤，其对算法有更多控制权
* 策略模式每个策略需要定义整个算法的完整实现，但是更有弹性
